## 4. Verification and Status Discipline

**CRITICAL DISTINCTION:**

| Status Level | Definition |
|--------------|------------|
| **Implementation Complete** | File edited, committed, pushed |
| **Live Verification Pending** | Deployment/propagation not yet confirmed |
| **Complete and Verified** | Live path loads correctly, rendered result confirmed usable |

**Rule:**
- A website change is ONLY "Complete and Verified" when live rendered result is confirmed usable at intended path
- Committed and pushed is NOT enough
- Verification is mandatory before claiming completion

**Allowed Status Labels:**
- ✅ Complete and verified
- ⚠️ Partially complete
- 🔧 Change made but breakage found
- ↩️ Rolled back
- ❓ Needs manual review

**Prohibited:** Vague success language like "done" without verification
