# Howard Prime Telegram — Publishing Checklist

## Document Purpose
Pre-post checklist to prevent errors, maintain quality, and ensure brand consistency.

**Use this checklist before EVERY Telegram post.**

---

## Section 1: Content Verification

### Accuracy Check
- [ ] All facts verified against source material
- [ ] Dates correct and consistent
- [ ] Commit hashes accurate (if referenced)
- [ ] Links tested and functional
- [ ] No fabricated claims or quotes

### Source Attribution
- [ ] External news properly attributed
- [ ] Direct quotes marked as such
- [ ] Images credited if not Howard/owned
- [ ] Audio sources noted if applicable

---

## Section 2: Format Compliance

### Caption Format
- [ ] Opening line follows headline format
- [ ] Length under 1,500 characters (preferably under 800)
- [ ] Line breaks used correctly (single between paragraphs)
- [ ] Link on its own line at end

### Typography
- [ ] Howard sign-off present: `— Howard` (em-dash)
- [ ] No more than 2 emojis total
- [ ] No exclamation marks (unless humor with approval)
- [ ] Consistent spacing throughout

### Link Handling
- [ ] URL is correct and complete
- [ ] Link preview generates correctly (or disabled intentionally)
- [ ] Shortened URLs don't obscure destination
- [ ] UTM parameters added if tracking required

---

## Section 3: Image Verification

### Image Selection
- [ ] Correct tier for content type (see Branding Rules)
- [ ] Image not overused (check last 10 posts)
- [ ] Emotional register matches caption
- [ ] Quality sufficient (not pixelated or cropped awkwardly)

### Image File Check
- [ ] Correct file selected
- [ ] File size under 5MB
- [ ] Dimensions appropriate (min 800x450, pref 1280x720)
- [ ] No unintended text/watermarks

### Special Image Rules
- [ ] **If using howard-canonical-portrait.jpg:** Gary approval obtained
- [ ] **If humor post:** Image is from approved humor set
- [ ] **If news post:** Howard image is newsroom context

---

## Section 4: Audio Verification (If Applicable)

### Audio File Check
- [ ] File plays correctly
- [ ] Length appropriate (30-90 sec for humor, 1-3 min for updates)
- [ ] Howard voice consistent (voice_id: IKne3meq5aSn9XLyUdCD)
- [ ] Audio matches script/caption

### Audio Presentation
- [ ] 🎧 emoji included in caption
- [ ] Audio link or file attached
- [ ] Thumbnail set (Howard image for consistency)

---

## Section 5: Timing and Pacing

### Content Type Timing
- [ ] Daily update: Posted ~03:00 AEST
- [ ] News briefing: Appropriate urgency level
- [ ] Humor: 48+ hours since last humor post
- [ ] Backfill: Scheduled, not interrupting current content

### Gap Check
- [ ] No posts within 2 hours (unless breaking news)
- [ ] Humor not conflicting with serious news
- [ ] Not flooding channel (max 3 posts/day)

---

## Section 6: Content Type Specific Checks

### Daily Update Posts
- [ ] Time period specified
- [ ] 3-4 achievements listed
- [ ] Evidence provided (commits, metrics)
- [ ] "Decisions Made" section included

### News Briefing Posts
- [ ] Headline structure correct
- [ ] "Why This Matters" framing present
- [ ] Reality check included
- [ ] Source attribution correct

### Philosophy Posts
- [ ] Thesis clear in opening
- [ ] Sections well-structured
- [ ] Conclusion meaningful
- [ ] Tone: thoughtful, authoritative

### Comic Relief Posts
- [ ] **HUMOR QUOTA CHECK:** Max 1 today
- [ ] Context image appropriate
- [ ] Tone: dry, observational
- [ ] Not mean-spirited or punching down

### Branding Posts
- [ ] **APPROVAL CHECK:** Gary signed off
- [ ] Canonical portrait image used
- [ ] Identity clear in first paragraph
- [ ] Call to action present

---

## Section 7: Brand Consistency

### Voice Check
- [ ] Tone appropriate for content type
- [ ] Howard personality consistent
- [ ] No jarring shifts in register
- [ ] No imitation of other voices/accounts

### Visual Consistency
- [ ] Image style matches content
- [ ] Color treatment appropriate
- [ ] No low-quality or off-brand visuals

### Strategic Alignment
- [ ] Post supports channel purpose
- [ ] Content valuable to target audience
- [ ] Not filler or "posting to post"

---

## Section 8: Technical Readiness

### Platform Check
- [ ] Telegram client accessible
- [ ] Channel selected correctly (howard_prime)
- [ ] Posting permissions confirmed
- [ ] No scheduled maintenance conflicts

### Asset Preparation
- [ ] All images uploaded/ready
- [ ] Audio files accessible
- [ ] Links copied correctly
- [ ] Caption drafted and reviewed

### Rollback Plan
- [ ] Edit capability available if needed
- [ ] Deletion plan if critical error
- [ ] Follow-up post drafted if correction needed

## Final Approval Sign-Off

**Before clicking send:**

- [ ] Full checklist reviewed
- [ ] Content type confirmed
- [ ] Image verified against Branding Rules
- [ ] Timing appropriate
- [ ] No checklist items failed

**If ANY item failed:**
- Fix the issue, OR
- Escalate to Gary for decision

**Never post with known violations.**

---

## Post-Publication Verification

### Immediate (within 5 minutes)
- [ ] Post renders correctly on mobile
- [ ] Images display properly
- [ ] Links work when tapped
- [ ] Audio plays (if applicable)
- [ ] No formatting errors visible

### Within 1 Hour
- [ ] No error reports from subscribers
- [ ] Engagement appropriate for content type
- [ ] No technical issues reported

### Documentation
- [ ] Post logged in publishing record
- [ ] Any issues noted for future reference
- [ ] Success/failure patterns tracked

---

## Emergency Procedures

### If Posted in Error
1. **Assess severity:** Minor typo vs major error
2. **Typo/Minor:** Edit if possible, comment correction
3. **Major error:** Delete and repost corrected version
4. **Document:** Note what happened for process improvement

### If Technical Issue
1. **Don't panic**
2. **Document:** Screenshot the issue
3. **Escalate:** Notify Gary immediately
4. **Communicate:** If delay expected, inform subscribers
