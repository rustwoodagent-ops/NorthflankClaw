# Morning Review Checklist

**Purpose:** Daily operational review template for rustwood.au business
**Used by:** Aaron + Howard

---

## Pre-Review (60 seconds)

- [ ] Check calendar for meetings/deadlines
- [ ] Review yesterday's MEMORY.md entry
- [ ] Check for any urgent messages/notifications

---

## Part 1: Storefront Status (2 min)

- [ ] Review Storefront State Matrix — any status changes?
- [ ] Check Fulfilment Readiness Checklist — blockers still valid?
- [ ] Verify Launch Blocker Tracker — ETA updates needed?

**Questions:**
- Are any products closer to launch today?
- Any new blockers emerged?
- Should priorities shift?

---

## Part 2: Page Health (2 min)

- [ ] Any verification failures from automated checks?
- [ ] Manual spot-check of critical pages (if changes made yesterday)
- [ ] Review Page Verification Checklist — update if verified

**Quick Checks:**
- Homepage loads?
- Howard Operator OS page loads?
- No broken images or links?
- Mobile view acceptable?

---

## Part 3: Revenue & Conversion (2 min)

- [ ] Any sales overnight? (when store live)
- [ ] Any new email signups? (when capture live)
- [ ] Any support requests or inquiries?
- [ ] Check waitlist growth (if applicable)

**Questions:**
- What's the lead flow today?
- Any conversion issues reported?

---

## Part 4: Active Work Review (3 min)

- [ ] What was committed yesterday?
- [ ] What's in progress today?
- [ ] Any blockers requiring human input?
- [ ] What's the highest-leverage next move?

**Priority Check:**
- Tier 1 (Revenue) work happening?
- Tier 2-3 getting appropriate attention?
- Tier 5 not consuming prime cycles?

---

## Part 5: Quick Planning (2 min)

- [ ] Confirm today's top 3 priorities
- [ ] Check for any scheduled meetings or calls
- [ ] Note any time-sensitive tasks
- [ ] Update MEMORY.md with today's focus

---

## Post-Review Actions

- [ ] Log today's review in memory/YYYY-MM-DD.md
- [ ] Update any tracker files with new info
- [ ] Queue any human-input requests
- [ ] Set status: Ready to execute

---

## Emergency Escalation Triggers

**Escalate immediately if:**
- 🔴 Site is down
- 🔴 Store/checkout broken (when live)
- 🔴 Payment processing failing
- 🔴 Customer complaint/issue
- 🔴 Security concern

---

## Time Budget

| Section | Time |
|---------|------|
| Pre-review | 1 min |
| Storefront status | 2 min |
| Page health | 2 min |
| Revenue/conversion | 2 min |
| Active work | 3 min |
| Quick planning | 2 min |
| **Total** | **~12 minutes** |

---

## Weekly Additions (Fridays)

- [ ] Review week in MEMORY.md
- [ ] Check Launch Blocker progress for the week
- [ ] Plan next week's priorities
- [ ] Identify any repeating friction
- [ ] Update SOUL.md if lessons learned
