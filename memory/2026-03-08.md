## Twitter/X (xurl) auth troubleshooting notes (21:00 AEST)
- `xurl` installed via npm and available at `/usr/bin/xurl` (version observed: **1.0.3**).
- Issue: `xurl auth oauth2` appeared to hang (cursor dropping line / blinking) in WSL terminal; also encountered `bind: address already in use` on `127.0.0.1:8080` due to a stuck prior xurl listener.
- Fix: kill stuck listener process holding 8080 (e.g. `lsof -i :8080` then `kill <pid>` or `pkill xurl`) and retry.
- Critical prerequisite discovered: `xurl auth status` reported **“No apps registered. Use 'xurl auth apps add' to register one.”**
- Therefore OAuth2 will not proceed until an X Developer app is registered locally with client credentials (must be entered manually by Aaron outside chat): `xurl auth apps add <name> --client-id ... --client-secret ...`, then rerun `xurl auth oauth2`.
- WSL2 may not auto-open browser; user may need to copy the printed auth URL into Windows browser to complete OAuth.
